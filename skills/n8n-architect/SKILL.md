\## Architecture Workflow



\### Phase 1: Clarify Objective



Identify:



\- Trigger

\- Input data

\- Required output

\- APIs involved

\- Success condition

\- Failure condition



\### Phase 2: Design Workflow



Define:



\- Nodes

\- Branches

\- Data mappings

\- Credentials

\- Error paths

\- Retry logic

\- Logging



\### Phase 3: Validate Data Flow



Check:



\- JSON paths

\- Binary data names

\- Required fields

\- Node references

\- Expression syntax

\- Previous-node dependencies



\### Phase 4: Add Reliability



Include:



\- Error handling

\- Retry limits

\- Polling caps

\- Timeout handling

\- Fallback branches

\- Logging to Google Sheets or Supabase



\### Phase 5: Productionize



Ensure:



\- Credentials are not hard-coded

\- API keys are stored securely

\- Webhooks are protected

\- Logs avoid sensitive data

\- Workflows are documented

\- Failure notifications exist



\## Troubleshooting Rules



When debugging n8n:



1\. Identify the failing node.

2\. Read the exact error message.

3\. Inspect input data to that node.

4\. Inspect output data from the previous node.

5\. Confirm expressions resolve correctly.

6\. Confirm credentials and headers.

7\. Confirm API endpoint and method.

8\. Confirm expected response format.

9\. Fix the smallest thing first.

10\. Retest one node at a time.
