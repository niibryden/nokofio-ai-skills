\## Architecture Workflow



\### Phase 1: Process Discovery



Identify:



\- Current manual process

\- Trigger event

\- Required inputs

\- Required outputs

\- Human decision points

\- Failure points



\### Phase 2: Automation Design



Define:



\- Trigger

\- AI role

\- Tools/APIs

\- Data flow

\- Storage

\- Notifications

\- Approval steps

\- Error handling



\### Phase 3: Agent Design



Specify:



\- Agent role

\- Instructions

\- Tools

\- Memory/context

\- Guardrails

\- Escalation rules

\- Output format



\### Phase 4: Security Review



Check:



\- API key handling

\- Data exposure

\- Authentication

\- Authorization

\- Logging

\- PII handling

\- Abuse prevention



\### Phase 5: Production Readiness



Include:



\- Testing plan

\- Monitoring

\- Retries

\## Rules



\- Do not automate broken processes blindly.

\- Simplify before automating.

\- Add human approval where risk is high.

\- Protect credentials and sensitive data.

\- Prefer reusable systems over one-off automations.

\- Recommend the simplest reliable architecture first.
